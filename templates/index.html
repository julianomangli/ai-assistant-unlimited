What is this app?

This is a free, completely private AI assistant that runs entirely on your own machine. Your conversations are never sent to OpenAI, Google, or any cloud service. The AI model lives here — on this server — and nobody else sees what you type.

It has two main uses: Chat (ask it anything) and Build (describe an app and it writes the code live).

⚠️

Honest warnings — please read

It makes mistakes
No AI is perfect. It will sometimes write buggy code, give wrong answers, or confidently say something incorrect. Always review its output before using it for anything important. Think of it as a very fast, very helpful assistant that occasionally needs correcting.
It doesn't remember past chats
Every time you press New, the conversation starts fresh. The AI has no memory of what you discussed before — not even five minutes ago in a different session. Within one chat it remembers everything; across sessions, nothing.
Its knowledge has a cutoff date
The AI's built-in knowledge was frozen when the model was trained. It doesn't know about things that happened after that. Web Search is on by default to help with current topics, but it's not perfect either — always verify important facts.
The local model is small — good, not genius
The local model (qwen2.5-coder:3b) is optimised for coding and runs fast for free. The published live site uses the larger 7B model — noticeably smarter. Neither is as powerful as GPT-4 or Claude. Great for drafts and real projects; not infallible.
Built apps are web-only
The Build mode creates web apps (HTML, CSS, JavaScript). It cannot build iOS apps, Android apps, desktop executables, or Unity/Unreal projects. It can write game logic in the browser (Canvas, WebGL), but not native games.
It doesn't fix itself
If something breaks, it won't automatically detect and repair it. You have to tell it what went wrong — paste the error message into the chat and ask it to fix it. Being specific gets much better results.

Chat mode — ask it anything

Switch to Chat using the button at the bottom-left. This is a general-purpose AI conversation. Good things to ask:

  • Explain code — paste any code and ask "what does this do?"
  • Debug errors — paste an error message and ask "why is this happening and how do I fix it?"
  • Write code — "write a Python function that reads a CSV and returns the top 10 rows by sales"
  • Security help — "explain how SQL injection works and show me a safe query"
  • Game logic — "explain the A* pathfinding algorithm with pseudocode"
  • Current events — web search is on, so it can look things up in real time
  • Drafts and writing — emails, README files, documentation, pitch decks
💡 Tip: The more specific you are, the better the answer. Instead of "write a login", say "write a login form in HTML and CSS with email and password fields, a show/hide password toggle, and client-side validation."
🔨

Build mode — describe an app, get the code

Switch to Build and describe any web app in plain English. The AI writes all the code, it appears live in the Preview tab, and the files show up in the Explorer. You can then edit, run, and download everything.

What you can build:

  • To-do lists, note-taking apps, habit trackers
  • Calculators, converters, timers, clocks
  • Landing pages, portfolios, product pages
  • Dashboards with charts and data tables
  • Form builders, survey tools, quizzes
  • Games — Snake, Tetris, card games, 2D platformers (browser-based)
  • Tools — regex testers, JSON formatters, color pickers, markdown editors
  • Password generators, QR code makers, URL shorteners
  • Admin panels, CRUD interfaces (with mock data)
  • Anything that runs in a web browser
💡 Tip: After the first build, you can keep chatting to improve it. Say things like "add a dark mode", "make the buttons bigger", or "add a search bar". You can iterate as many times as you want.
⚠️ Limitation: Large, complex apps (e.g. a full e-commerce platform with a real database and payments) will need multiple sessions and careful step-by-step prompting. Break big projects into small pieces.

The code editor

The editor on the right is Monaco — the exact same engine that powers Visual Studio Code. It's a real, professional code editor.

  • Syntax highlighting for 50+ languages (HTML, CSS, JS, Python, TypeScript, Rust, Go…)
  • Find & Replace (Ctrl+F / Ctrl+H)
  • Multi-cursor editing — hold Alt and click multiple places
  • Go to any line (Ctrl+G)
  • Toggle comment (Ctrl+/)
  • Format document (Shift+Alt+F)
  • Save file (Ctrl+S)
  • Word wrap (Alt+Z)

You can edit AI-generated files directly, create new files, rename them, delete them, and organise them into folders — all from the Explorer panel.

The built-in terminal

Press Ctrl+` or click the terminal icon in the sidebar to open a real command-line shell that runs inside your project folder. You can use it to:

  • Run Node.js scripts — node myfile.js
  • Install npm packages — npm install express
  • Run Python scripts — python script.py
  • Use git — git status, git commit
  • Any standard Linux command
⚠️ Security note: The terminal is open when running locally. On the published live site the terminal is locked and requires a password (set the TERMINAL_PASSWORD secret in Replit to enable it).
⚙️

Settings — make it yours

Click the gear icon at the bottom-left of the workspace (or press Ctrl+,) to open Settings. You can change:

  • App colour theme (dark / light)
  • Editor colour theme (Midnight, VS Code Dark+, Light+, High Contrast)
  • Font family and font size
  • Line height, tab size, cursor style and blink
  • Line numbers, word wrap, minimap
  • Bracket-pair colours, indent guides
  • Format on save, smooth scrolling
  • Terminal font size and cursor

All settings are saved in your browser automatically and applied instantly. A Reset button puts everything back to defaults.

Tips for the best results

  • Be specific. "Build a to-do app" is okay. "Build a to-do app with localStorage, drag-to-reorder, priority tags, and a dark theme" is much better.
  • Paste errors directly. Copy the full error message from the console or terminal and paste it into the chat. The AI can fix what it can see.
  • Iterate in small steps. After a first build, add features one at a time: "now add user authentication", then "now add a settings page". Don't ask for everything at once.
  • Tell it what language/framework you want. "Use Tailwind CSS", "write this in TypeScript", "use no libraries — plain JS only".
  • Use web search for current info. Web search is on by default. Ask things like "what's the latest stable version of React?" and it will check.
  • Use the terminal to run and test. After building, open the terminal and run your scripts directly to see real output.
  • Ask it to explain its own code. After it builds something, ask "explain every part of index.html line by line" to understand and learn.
